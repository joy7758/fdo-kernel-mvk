# FDO Relation Note

MVK should be interpreted as a verification and conformance surface rather than a standalone object profile.

## FDO-facing interpretation

MVK contributes execution integrity verification that can support profile validation, replay checking, and tamper detection.

## What MVK contributes

- deterministic execution checks
- replay validation
- tamper-detection surface
- conformance-oriented verification path

## What MVK does not yet claim

- standalone FDO object profile
- registry profile status
- final normative specification

## Current architectural role

MVK is the Execution Integrity Verification Entry in the Digital Biosphere Architecture.
